[GenAI] 各 key 的作用

  This is a corporate/enterprise OpenAI proxy setup (not direct OpenAI API), designed for environments where you authenticate through an enterprise gateway:

  ┌─────────────────────────┬───────────────────────────────────────────────────────────────────────────────────┐
  │           Key           │                                       作用                                        │
  ├─────────────────────────┼───────────────────────────────────────────────────────────────────────────────────┤
  │ openai_token 【-】        │ 预设的 Bearer token，如果有值则直接用，跳过 token 交换                            │
  ├─────────────────────────┼───────────────────────────────────────────────────────────────────────────────────┤
  │ openai_token_url        │ Token 交换端点 — 用 username/password 换取 JWT token（企业 SSO 网关）             │
  ├─────────────────────────┼───────────────────────────────────────────────────────────────────────────────────┤
  │ openai_health_url       │ 健康检查端点，检测 AI 服务是否可用                                                │
  ├─────────────────────────┼───────────────────────────────────────────────────────────────────────────────────┤
  │ openai_mm_url 【-】      │ 多模态 API 端点 — 实际发送聊天/图片请求的 URL（类似 OpenAI 的 /chat/completions） │
  ├─────────────────────────┼───────────────────────────────────────────────────────────────────────────────────┤
  │ openai_chat_url         │ 聊天 API 端点（代码里没用到，可能是备用）                                         │
  ├─────────────────────────┼───────────────────────────────────────────────────────────────────────────────────┤
  │ openai_user_name        │ 企业网关认证用户名                                                                │
  ├─────────────────────────┼───────────────────────────────────────────────────────────────────────────────────┤
  │ openai_password         │ 企业网关认证密码                                                                  │
  ├─────────────────────────┼───────────────────────────────────────────────────────────────────────────────────┤
  │ openai_application_id   │ 企业平台分配的应用 ID（请求头 GAI-Platform-Application-ID）                       │
  ├─────────────────────────┼───────────────────────────────────────────────────────────────────────────────────┤
  │ openai_application_name │ 应用名称（未在代码中使用）                                                        │
  ├─────────────────────────┼───────────────────────────────────────────────────────────────────────────────────┤
  │ head_token_key  【-】    │ token 放在哪个 HTTP Header 里，默认 Authorization                                 │
  └─────────────────────────┴───────────────────────────────────────────────────────────────────────────────────┘

  简单说：这是为公司内网的 AI 代理网关设计的，先用账号密码换 JWT token，再用 token 调用内部的 OpenAI 兼容 API。
